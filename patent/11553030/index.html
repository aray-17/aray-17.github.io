Service worker configured to serve multiple single page applications

Microsoft Technology Licensing LLC

A single service worker that handles offline, caching, and routing for multiple single-page apps on the same website — instead of each app registering its own.

Instead of binding to a single app, the service worker is bound to the root of the website. A router and routing table inside the worker decide which app each incoming URL belongs to and dispatch it to that app's controller.

Multiple single-page apps living on the same site share one offline / caching / routing layer, instead of each registering and maintaining its own service worker.
