A single composite request lets a client app talk to multiple backend services at once, with each service streaming its response back on the same connection.

When a client sends a composite request that names actions across two or more backend services, the framework splits the request, dispatches each part to the right service, and streams responses back as separate transmissions on the same connection.

The app reads from one unified surface and doesn't have to orchestrate the fan-out itself. Each piece can start rendering as soon as it arrives, so heavy parts of the request don't block the lighter ones.
