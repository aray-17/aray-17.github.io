Agent Capsules: Quality-Gated Granularity Control for Multi-Agent LLM Pipelines

Author
Aninda Ray
Identifier
arXiv:2605.00410v1
Categories
cs.CL, cs.AI
Submitted
May 2026

A 14-agent pipeline runs at roughly half the input tokens with quality intact — by knowing when to step back from compound execution toward per-agent dispatch, rather than rewriting merged prompts.

Multi-agent LLM pipelines normally issue one call per agent. Compound execution merges agents into fewer calls to save tokens, but naive merging silently degrades quality through tool loss and prompt compression.

Agent Capsules treats execution as an optimization problem with empirical quality constraints. The runtime instruments coordination overhead per group, scores composition opportunity, picks among three execution modes (standard compound, two-phase compound, or sequential per-agent), and gates every mode switch on rolling-mean output quality. A controlled negative result confirms that injecting more context into a merged call worsens compression rather than relieving it — so the escalation ladder recovers quality by moving toward per-agent dispatch rather than rewriting merged prompts.

  • 51% fewer fine-mode input tokens vs. hand-crafted 14-agent LangGraph, at +0.020 quality
  • 68% fewer tokens than MIPROv2 on a 5-agent DSPy pipeline, at +0.052 quality
  • 0 per-pipeline engineering or training data required

Even before compound mode fires, the runtime matches both hand-tuned and compile-time baselines through automatic policy resolution, cache-aligned prompts, and topology-aware context injection — without per-model configuration.

In practice

  • For developers building multi-agent LLM pipelines: a drop-in replacement for hand-tuned orchestration. Write the pipeline once in a compact DSL; the runtime delivers improved quality and explicit control over the cost-versus-quality tradeoff.
  • For cloud providers and platform teams: cost optimizations they can trust. Per-tenant quality floors gate every mode switch, so capacity freed from naive compound execution never dips below an agreed-on quality bar.
  • For research: an empirical demonstration that escalating down to per-agent dispatch beats engineering merged-call prompts when output quality drops.
